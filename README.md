# NFC 门禁项目

一个基于 Arduino UNO、RC522 和 MG996R 舵机的 NFC 门禁控制示例。

项目初衷为给我的本科宿舍门装一个简单的 NFC 门禁系统，省去平时带钥匙的麻烦。这个项目思路来源于 https://github.com/vaaandark/smart-door 。代码用 AI 完全重写了一遍，支持写卡和删卡功能，把卡信息持久化到 EEPROM 里，断电不丢失。零件选购可以参考上述链接里的 README。

## 功能

- 读取 RC522 IC 卡 UID。
- 白名单持久化到 UNO 内置 EEPROM，断电不丢失。
- 刷到已登记卡后控制 MG996R 舵机开门。
- 支持写卡模式和删卡模式。
- 写卡成功、删卡成功时，舵机都会给出短暂反馈动作。

## 硬件连接

完全参考 https://github.com/vaaandark/smart-door 里的接线图即可，此外保留一根公对公杜邦线用于写/删卡。

### 写/删卡模式引脚

当前固件里使用一个公共低电平源和两个感应脚来切换模式：

- D4 -> 低电平源输出
- D2 -> 写卡模式感应脚
- D3 -> 删卡模式感应脚

使用方式：

- 短接 D4 和 D2，进入写卡模式
- 短接 D4 和 D3，进入删卡模式
- 不短接时，进入正常刷卡开门模式

如果写卡和删卡同时满足，当前程序优先处理写卡模式。

## 工作流程

### 正常模式

- 刷到已登记卡：舵机开门 4 秒后自动关门。
- 刷到未登记卡：拒绝访问。

### 写卡模式

- 短接 D4 和 D2。
- 刷一张新卡，卡的 UID 会写入 EEPROM 白名单。
- 如果卡已存在，会直接忽略重复写入。
- 白名单最多保存 20 张卡。

### 删卡模式

- 短接 D4 和 D3。
- 刷一张已登记卡，系统会从 EEPROM 白名单中删除该卡。
- 删除后，后面的卡会向前移动，保持列表连续。

## 编译与上传

本工程使用 PlatformIO。

1. 安装 PlatformIO IDE 或 PlatformIO CLI。
2. 打开项目目录。
3. 执行构建：

```bash
pio run
```

4. 上传到 UNO：

```bash
pio run --target upload
```

5. 打开串口监视器：

```bash
pio device monitor
```

## 现有参数

- 最大白名单数量：20
- 舵机关闭角度：0
- 舵机打开角度：180
- 开门保持时间：4 秒

## 备注

- 代码里的卡号是按 UID 做白名单，不读取卡内扇区数据。
- 舵机动作和 RC522 读卡都需要稳定供电，否则容易出现抖动或读卡失败。
